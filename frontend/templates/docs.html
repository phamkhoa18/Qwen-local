VKS Legal AI API Reference

Tài liệu kỹ thuật đầy đủ để kết nối, tích hợp API Trợ lý Pháp luật AI cho Viện Kiểm Sát Nhân Dân Việt Nam.

Tổng Quan

VKS Legal AI cung cấp REST API tương thích chuẩn OpenAI, hỗ trợ tra cứu pháp luật Việt Nam với hệ thống Agentic RAG (Hybrid Search + Multi-step Reasoning).

🔍

Hybrid Search

BM25 keyword + FAISS vector + RRF Fusion cho kết quả chính xác nhất.

🤖

Agentic RAG

Agent tự phân loại, tìm kiếm đa bước, tham chiếu chéo điều luật.

🌐

Multi-LLM

Qwen3 local + OpenRouter cloud (Claude, GPT-4o, Gemini).

📡

OpenAI Compatible

Tích hợp ngay với bất kỳ client OpenAI nào có sẵn.

ℹ️
Base URL: Tất cả API đều bắt đầu bằng domain/IP của server. Ví dụ: https://your-server.com/v1/chat/completions

Xác Thực (Authentication)

Tất cả API yêu cầu API Key trong header:

HTTP Header
Authorization: Bearer vks-xxxxxxxxxxxx

API Key được tạo qua giao diện quản trị hoặc API admin. Xem phần Admin API để biết cách tạo key.

Quick Start

Lấy API Key

Truy cập giao diện chính → Quản lý API Keys → Tạo khóa mới

Gửi request đầu tiên

Dùng cURL, Python hoặc bất kỳ HTTP client nào

Nhận kết quả

Câu trả lời pháp luật kèm trích dẫn nguồn chính xác

cURL
curl -X POST https://your-server.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer vks-YOUR_API_KEY" \
  -d '{
    "model": "qwen3:30b-a3b",
    "messages": [{"role": "user", "content": "Phân tích Điều 173 BLHS 2015"}],
    "use_rag": true,
    "stream": false
  }'

Chat Completions

POST /v1/chat/completions Hỏi đáp pháp luật / Chat AI

Endpoint chính cho chat. Hỗ trợ 2 chế độ:

  • use_rag: true — Tra cứu pháp luật (Agentic RAG + Hybrid Search)
  • use_rag: false — Chat AI trực tiếp (không RAG)

Request Body

Tham sốKiểuBắt buộcMô tả
modelstringrequiredID model (vd: qwen3:30b-a3b)
messagesarrayrequiredMảng tin nhắn [{role, content}]
use_ragbooleanoptionalBật tra cứu pháp luật (default: true)
streambooleanoptionalStreaming response (default: false)
temperaturefloatoptional0.0-2.0 (default: 0.3)
max_tokensintegeroptionalSố token tối đa (default: 8192)

Response (non-streaming)

JSON
{
  "id": "chatcmpl-abc123",
  "model": "qwen3:30b-a3b",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Theo Điều 173 BLHS 2015..."
    },
    "finish_reason": "stop"
  }],
  "sources": [
    {
      "title": "Bộ luật Hình sự 2015",
      "article": "Điều 173",
      "content": "Tội trộm cắp tài sản...",
      "score": 0.89
    }
  ],
  "agent": {
    "intent": "LOOKUP",

Document Search

POST /v1/documents/search Tìm kiếm văn bản pháp luật (không qua LLM)

Tìm kiếm nhanh trong 178K+ văn bản pháp luật bằng Hybrid Search (BM25 + Vector). Không gọi LLM, trả kết quả ngay.

cURL
curl -X POST /v1/documents/search \
  -H "Authorization: Bearer vks-YOUR_KEY" \
  -d '{"messages": [{"role": "user", "content": "Điều 173 BLHS"}]}'

List Models

GET /v1/models Danh sách model local + cloud

Trả về danh sách model từ Ollama (local) và OpenRouter (cloud). Không cần authentication.

Response
{
  "object": "list",
  "data": [
    {"id": "qwen3:30b-a3b", "owned_by": "ollama-local", "provider": "local"},
    {"id": "google/gemini-2.5-flash", "owned_by": "openrouter-google", "provider": "cloud"}
  ]
}

Admin API

POST /admin/login Đăng nhập admin, lấy JWT token
cURL
curl -X POST /admin/login \
  -d '{"username": "admin", "password": "your_password"}'
GET /admin/health Health check (Ollama, MongoDB, RAG, Agent)
Response
{
  "status": "ok",
  "ollama_connected": true,
  "rag_ready": true,
  "total_legal_chunks": 178000,
  "openrouter_configured": true,
  "agent_enabled": true,
  "search_mode": "hybrid (FAISS + BM25 + RRF)"
}
POST /admin/api-keys Tạo API key mới

Yêu cầu JWT token (từ /admin/login).

cURL
curl -X POST /admin/api-keys \
  -H "Authorization: Bearer JWT_TOKEN" \
  -d '{"name": "My App", "rate_limit": 60}'

Tích Hợp Python

Python
import requests

API_URL = "https://your-server.com"
API_KEY = "vks-YOUR_API_KEY"

response = requests.post(f"{API_URL}/v1/chat/completions", 
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={
        "model": "qwen3:30b-a3b",
        "messages": [{"role": "user", "content": "Điều 173 BLHS 2015?"}],
        "use_rag": True
    })

data = response.json()
print(data["choices"][0]["message"]["content"])

# In nguồn tham khảo
for src in data.get("sources", []):
    print(f"  📄 {src['title']} - {src['article']} ({src['score']:.0%})")

}

OpenAI SDK Compatible

Python — OpenAI SDK
from openai import OpenAI

client = OpenAI(
    api_key="vks-YOUR_API_KEY",
    base_url="https://your-server.com/v1"
)

response = client.chat.completions.create(
    model="qwen3:30b-a3b",
    messages=[{"role": "user", "content": "Điều 173 BLHS 2015?"}],
    extra_body={"use_rag": True}
)

print(response.choices[0].message.content)

Error Handling

StatusÝ nghĩaXử lý
200Thành công—
401API key không hợp lệKiểm tra lại API key
429Rate limit exceededChờ 60s hoặc nâng limit
500Server errorKiểm tra Ollama/MongoDB status

Rate Limiting

Mỗi API key có giới hạn request/phút (mặc định: 30 req/min). Có thể tùy chỉnh khi tạo key.

⚠️
Khi vượt rate limit, API trả về HTTP 429. Chờ 60 giây hoặc liên hệ admin để nâng giới hạn.
